📋 1. Tổng quan

🔐 Lưu ý quan trọng:
  • Tài khoản được cấp bởi admin — không có tính năng đăng ký công khai
  • Quên mật khẩu: Gửi email reset password với token JWT ngắn hạn (15 phút)
  • UUID v7 làm khóa chính cho tất cả entity
  • Validation được thực hiện ở DTO level (Jakarta Validation)
  • Phân quyền chi tiết theo API và phạm vi dữ liệu
  • File phân loại PUBLIC / PRIVATE (có quyền mới xem được)

Backend được thiết kế theo kiến trúc Domain-Driven Design (DDD) với các module riêng biệt. Sử dụng Spring Data JPA với PostgreSQL, Flyway cho migration, Spring Security với JWT cho xác thực và phân quyền.

🧬 4. Base Entity

📄 BaseEntity Base
Entity cơ sở cho tất cả entity trong hệ thống. Sử dụng UUID v7 làm khóa chính. Tự động audit createdAt, updatedAt, createdBy, updatedBy.
Các trường:
id: UUID • createdAt: LocalDateTime • updatedAt: LocalDateTime • createdBy: UUID • updatedBy: UUID • version: Long (Optimistic Locking) • isActive: Boolean (default: true) • isDeleted: Boolean (default: false)
📄 BaseEntityWithName Base
Mở rộng BaseEntity, thêm các trường name, code, description, slug. Dùng cho các entity cần tên hiển thị.
Các trường thêm:
name: String (nullable: false) • code: String • description: String (TEXT) • slug: String (unique)
📄 BaseEntityWithSort Base
Mở rộng BaseEntityWithName, thêm trường sortOrder và isFeatured. Dùng cho các entity cần sắp xếp hoặc đánh dấu nổi bật.
Các trường thêm:
sortOrder: Integer (default: 0) • isFeatured: Boolean (default: false)
📝 Code BaseEntity mẫu:
@Getter
@Setter
@MappedSuperclass
@EntityListeners(AuditingEntityListener.class)
public abstract class BaseEntity {

    @Id
    @GeneratedValue(strategy = GenerationType.UUID)
    @Column(columnDefinition = "UUID DEFAULT gen_random_uuid()")
    private UUID id;

    @CreatedDate
    private LocalDateTime createdAt;

    @LastModifiedDate
    private LocalDateTime updatedAt;

    @CreatedBy
    private UUID createdBy;

    @LastModifiedBy
    private UUID updatedBy;

    @Version
    private Long version = 0L;

    @Column(name = "is_active")
    private Boolean isActive = true;

    @Column(name = "is_deleted")
    private Boolean isDeleted = false;
}

📁 14. File Management

📁 File storage:
  • PUBLIC: /uploads/public/ — ai cũng xem được (avatar, ảnh công khai)
  • PRIVATE: /uploads/private/ — cần đăng nhập + quyền mới xem được
  • Ảnh upload tự động tạo 3 phiên bản: large (1920x1080), medium (1024x768), small (480x320)
  • File không phải ảnh được nén LZ4 để tiết kiệm dung lượng

🛡️ 15. Phân quyền chi tiết

15.1. Phân quyền theo Role

👑 SYSTEM_ADMIN
Toàn quyền trên hệ thống. Quản lý user, role, tất cả dữ liệu.

15.4. File Access Control

// Kiểm tra quyền truy cập file
public boolean canAccessFile(FileDescription file, User user) {
    switch (file.getVisibility()) {
        case PUBLIC:
            return true;
        case PROTECTED:
            return user != null && hasPrivilege(user, "FILE_VIEW_PROTECTED");
        case PRIVATE:
            return user != null && (
                user.getId().equals(file.getUploadedBy()) ||
                hasPrivilege(user, "FILE_VIEW_PRIVATE")
            );
        default:
            return false;
    }
}
